# Tour de Russie - Инструкция по запуску

## Быстрый старт

### 1. Установка зависимостей

```bash
npm install
```

### 2. Запуск локально для проверки

```bash
npm run dev
```

Откройте браузер: http://localhost:8080

### 3. Сборка для продакшена

```bash
npm run build
```

Готовые файлы появятся в папке `dist/`

### 4. Деплой на сервер

Загрузите содержимое папки `dist/` на ваш веб-сервер.

---

## Подробная инструкция

### Требования

- Node.js 18+
- npm или yarn

### Настройка переменных окружения

Для локальной разработки используйте `.env.local`:
```
VITE_API_URL=http://localhost:3000/api
VITE_FRONTEND_URL=http://localhost:8080
```

Для продакшена используйте `.env.production`:
```
VITE_API_URL=https://tourderussie.ru/api
VITE_FRONTEND_URL=https://tourderussie.ru
```

### Команды

```bash
npm run dev          # Запуск dev-сервера (localhost:8080)
npm run build        # Сборка для продакшена
npm run preview      # Предпросмотр собранного проекта
npm run lint         # Проверка кода
```

### Подключение к бекенду

Фронтенд использует API по адресу из `VITE_API_URL`.

Убедитесь, что бекенд:
1. Запущен и доступен
2. Настроен CORS для вашего домена
3. База данных PostgreSQL работает

### Деплой на сервер

#### Nginx конфигурация

```nginx
server {
    listen 80;
    server_name tourderussie.ru;
    root /path/to/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

#### Шаги деплоя

1. Соберите проект: `npm run build`
2. Загрузите содержимое `dist/` на сервер
3. Настройте nginx (см. выше)
4. Перезапустите nginx: `sudo systemctl restart nginx`

### Troubleshooting

**Проблема:** Не загружаются стили
- Проверьте, что все файлы из `dist/assets/` загружены

**Проблема:** API запросы не работают
- Проверьте `VITE_API_URL` в `.env.production`
- Проверьте, что бекенд запущен
- Проверьте CORS настройки на бекенде

**Проблема:** Ошибка 404 при переходе по прямой ссылке
- Настройте `try_files` в nginx (см. конфиг выше)

### Контакты

При возникновении проблем проверьте:
- Консоль браузера (F12 → Console)
- Логи nginx: `/var/log/nginx/error.log`
- Логи бекенда: `pm2 logs backend`
